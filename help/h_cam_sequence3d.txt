### H_CamSequence3D
> Inherits: Node3D < Node < Object

#### Description

This help document refers to [CamSequence3D] class! \
This node is only useful in combination with [H_VirtualCam3D] and [H_CineCam3D]. \
Attach [H_VirtualCam3D] nodes to this sequence. Then a [H_CineCam3D] can access it using various methods. \
Will automatically reload the internal array of virtual cameras on [constant Node.NOTIFICATION_CHILD_ORDER_CHANGED].

#### Signals
> * _signal_ index_changed(vcam3D : Object, index : int)()\
\
> Will be emitted, when the index of the currently selected virtual camera has changed. \
> Returns [H_VirtualCam3D] and the current index

#### Method Descriptions

> * _VirtualCam3D_ <a id="current_vcam">current_vcam()</a>\
\
> Returns the currently selected [H_VirtualCam3D]. \
> That means, that if the sequence is running, the target for the currently running blend is returned. \


> * _Array[VirtualCam3D]_ <a id="get_vcam3D_array">get_vcam3D_array()</a>\
\
> Returns all [H_VirtualCam3D] that are registred to this node. \
> This is basically the [H_VirtualCam3D] children of this node. \
> When virtual cams are added, removed or changed in order, the internal array will changed automatically. \
> You can use this method after such a change to get the new array. \


> * _VirtualCam3D_ <a id="vcam3D_at">vcam3D_at(index: int)</a>\
\
> Returns the [H_VirtualCam3D] at given index of the internal array. \

> * _void_ <a id="set_current_idx">set_current_idx(value : int)</a>\
